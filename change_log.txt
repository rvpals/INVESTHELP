## Android Changes

### Backup/Restore Overhaul — Generic Table-Based System (v6)
- Replaced hardcoded per-table export with generic `sqlite_master` table discovery
  - Queries `sqlite_master` for all user tables dynamically
  - Exports every row from every table as raw data
  - BLOB columns (e.g. logo) base64-encoded in JSON
  - New tables automatically included without code changes
- New backup format v6: `{"version":6,"tables":{"table_name":[{row},...],...}}`
- Restore: v6+ uses generic raw SQL `INSERT OR REPLACE` per table; v1-v5 uses legacy typed restore for backward compat
- Topological sort via `PRAGMA foreign_key_list` for FK-safe delete (children first) and insert (parents first) order
- All delete + insert operations wrapped in a single database transaction for atomicity
- Injected `DatabaseProvider` into `SettingsViewModel` for raw SQL access
- Auto-backup in `MainActivity.onStop()` also updated to use generic export
- Fixes: watch_lists, watch_list_items, definitions, sql_library, ai_library not restoring after uninstall+restore
- Also now backs up tables not previously included: csv_import_mappings, csv_named_mappings
